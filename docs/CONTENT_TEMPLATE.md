# Nodelog 기술 게시물 표준 템플릿 (v1 · 2026-07-03)

모든 신규 기술 게시물(자동 생성 포함)은 아래 구조를 따른다.
**외부 글 생성 파이프라인의 프롬프트에 이 템플릿을 반영할 것.**

## 필수 구조 (13단계)

1. **적용 상황** — 어떤 문제/에러가 발생했을 때 이 글을 쓰는지 (에러 메시지 원문 포함)
2. **적용 범위** — 제품·OS·버전 (예: PostgreSQL 14–16, Ubuntu 22.04)
3. **사전 확인·백업** — 작업 전 위험 요소와 백업 방법
4. **진단 순서** — 원인을 좁혀가는 단계별 확인 절차
5. **명령어/코드** — 복사해서 바로 실행 가능한 형태
6. **예상 정상 결과** — 각 명령이 성공했을 때의 출력 예
7. **실패 시 분기** — 예상과 다른 출력이 나올 때의 다음 단계
8. **영구 해결** — 근본 원인 제거 방법
9. **임시 조치의 위험** — 우회책을 쓸 때 감수하는 것
10. **재발 방지** — 모니터링·설정·프로세스 개선
11. **검증 환경** — 실제 확인한 환경. 확인하지 못했다면 "공식 문서 기준"임을 명시
12. **공식 참고 자료** — 실제 확인한 1차 출처만 (공식 문서·릴리스 노트·법령 원문)
13. **작성·검토 표기** — AI 초안 + 사람 검토 (사이트가 자동 표기)

## 금지 사항

- **확인 불가능한 경험 서술 금지**: "제가 직접 겪었습니다", "저희 팀에서는",
  "제가 본 장애의 80%", "직접 테스트한 결과" — 실제 검증 근거가 없으면 사용 금지.
  대신: "Redis 공식 문서와 일반적인 Linux 운영 환경 기준으로 가장 흔한 원인을 정리했습니다."
- **출처 없는 확정 수치 금지**: 벌금·과징금·비율·기한·법적 의무는 원문 출처 없이 확정 서술 금지
- **본문에 `#`(H1) 헤딩 금지** — 제목은 페이지가 렌더링. 본문은 `##`부터 시작
- **기존 글과 같은 주제 재생성 금지** — 발행 전 기존 제목과 대조 (API가 유사도 ≥0.5 자동 보류)
- 다른 사이트·공식 문서의 단순 요약 재작성 금지 — 아래 고유 가치 중 1개 이상 필수

## 고유 가치 (1개 이상 필수)

- 재현 가능한 테스트 절차
- 에러 원문별 의사결정표
- 실제 로그 해석 예시
- 버전별 동작 차이 정리
- 한국 환경(규정·인프라)에서의 적용 차이
- 직접 계산한 비교표

## 태그 규칙

- 태그는 개별 문자열로 (한 문자열에 여러 `#해시태그` 금지)
- 시리즈 태그: `series:이름`, 에피소드: `ep:N`
