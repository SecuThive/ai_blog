# GSC 검색 트래픽 보강 플레이북 (2026-07-15)

## 문제
콘텐츠·인프라·품질 게이트는 준비됐으나 **검색 유입이 사실상 0**(발행글 평균 조회 6.4, 대부분 오너 테스트). 궁극 병목은 순위·노출이며, 이는 코드가 아니라 **GSC 실적 데이터 기반 타게팅**으로만 풀린다.

## 왜 코드로 자동화 못 하나
백엔드에 Google OAuth(Gmail/Calendar/YouTube)는 있으나 **Search Console API 통합은 없음**. 실데이터는 오너가 export해야 함. (원하면 `webmasters.readonly` 스코프 추가로 자동 수집 파이프라인 구축 가능 — 별도 요청.)

## 1단계 — 데이터 확보 (오너, 5분)
1. [Search Console](https://search.google.com/search-console) → 속성 `www.thivelab.com` 선택
2. 좌측 **실적(Performance)** → 기간을 **최근 3개월**로
3. 우상단 **내보내기 → CSV 다운로드**(ZIP) → 압축 해제 (`Queries.csv`, `Pages.csv` 포함)
4. 해당 폴더를 저장소에 두거나 경로를 알려주면 분석 실행

## 2단계 — 근접키워드 분석 (자동)
```
node scripts/analyze-gsc.mjs <내려받은_GSC_export_폴더>
```
출력 3종:
- **① 근접키워드(8~20위, 노출≥5)** — 1페이지 문턱. **보강 1순위**. 이미 순위가 잡힌 글에 해당 키워드 섹션·비교표·FAQ를 더해 상위 5위권 진입.
- **② 상위노출인데 CTR<2%** — 순위는 좋은데 클릭이 안 됨 → **제목·메타디스크립션** 개선.
- **③ 노출 높은데 20위 밖** — 전용 글 부재 가능성 → **신규 글 기획**(엔진 content_planner에 키워드 전달).

## 3단계 — 보강 실행
| 유형 | 조치 | 담당 |
|---|---|---|
| ① 근접키워드 | 걸린 글(Pages.csv 대조) 본문에 키워드 H2·의사결정표·FAQ 추가, 내부링크 강화 | 수동/엔진 |
| ② 낮은 CTR | title 태그·메타디스크립션을 검색의도 맞춰 재작성 | 수동 |
| ③ 콘텐츠 갭 | 신규 글 — 엔진 프롬프트에 키워드 명시해 기획 유도 | 엔진 |

## 타이밍
- 발행 대부분이 5~6월 → 색인·순위 성숙 중. **노출 데이터가 쌓인 지금이 ① 분석 적기**.
- 근접키워드 보강 → 재크롤·재평가까지 1~2주 → 반복.
